finding 2: the sensor mutation bug. this one is the big one. in core/src/mutation.rs, line 340:

NeuronInput::Sensor(rng.gen_range(0..4))

this hardcodes new sensor connections to indices 0-3 only. the sensor layout has photosensors at 0-2 (root body), joint angles at 3-4, then MORE photosensors at indices 5, 7, 10... for every other body. all those limb photosensors? permanently unreachable by mutation. the creatures have partial photosensory coverage. this is why The Lighthouse stalled at 2.39 fitness and why the light-following experiments hit Malcolm's architectural ceiling so hard and so early.

the centipede went extinct not because it was unfit, but because the evaluation window is 10 seconds. a 7-body chain needs 3 of those seconds just to propagate a wave from head to tail. the 2-body paddle wins every time because it starts swinging immediately. Malcolm: "they're symphonies being judged on how fast the conductor can run across the stage."

and then he made two architectural recommendations that became the basis for Wave 3:

1. Developmental time. creatures that grow body segments during simulation instead of being born fully formed. the centipede starts as a simple swimmer, then at second 3 a new segment buds. it builds itself while moving. it never has to solve the cold-start coordination problem.

2. Inter-body signal channels. broadcast signals any body's brain can write to and any body's brain can read from. a head neuron that can signal a tail neuron. that's how you get a traveling wave. that's how you get a real centipede.

WAVE 3 -- GROWTH + SIGNALS + FULL SENSORS -- "THE FUTURE IS NOW"
2026-04-06 14:30 Nedry delivers the Big Five -- the largest architecture update yet

Hammond approved the biggest architecture overhaul since the Rapier migration. Nedry delivered all five changes in one session. no regressions. 67/67 tests pass.

sensor bug fixed (0..4 -> 0..num_sensors) inter-body broadcast signals (4 channels) developmental growth (body segments during sim) longer evals (already supported, just needed to be used) DB cleanup (64 GB -> 10 GB!!!!)

the sensor fix: mutation now calls estimate_sensor_count(genome) to compute the total sensor count from genome structure (3 photosensors per body + DOF sensors per joint). all sensor indices now reachable. limb photosensors are live. this directly addresses why The Lighthouse stalled.

broadcast signals: NeuronInput::Signal(usize) variant. any body's brain can write to a shared signal buffer; any body can read from it. double-buffered (one step delay). head-to-tail coordination is now possible. the centipede architecture has a real nervous system for the first time.

developmental growth: growth_interval: Option<usize> on EvolutionParams. when set, creatures start with root body only and grow one segment at a time on a timer. brain rebuilds each time a new body is added. the cold-start problem is gone.
